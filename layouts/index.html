Founders Note

Or Why we are solving API observability

A few years ago, I was a new hire at a food delivery company, and my role was the migration of a legacy service from PHP to Golang. I implemented the fields listed in the swagger docs, but because no one trusts swagger docs, I took some sample requests and compared the responses from the legacy service to the response of my new service. Everything matched. After weeks of implementation, I deployed, swapped the ingress to the new service and then proceeded to run tests to be sure everything worked as expected, and everything indeed worked as expected.But immediately I looked at our order monitoring charts, I knew I had messed up. Within 10mins, 20k orders lost. I was terrified. What happened?

Apparently in a few of the countries we ran in, a particular undocumented json response field was expected by the apps and I had no idea since these countries were not part of the samples I manually tested. We rolled back and fixed the issue, but at this point, I was even more pissed that there was no tool which could have prevented this problem.

Years later, this is that tool. Do you have similar war stories? We would love to hear them, so don't hesitate to leave a message. Thanks for being part of this journey.

Anthony (@tonialaribe) and Smile(@smilecs) (co-founders of APIToolkit)

What do you get?

Always up to date API docs

APIToolkit automatically listens to requests coming in and outside your services, and builds an understanding of what your API looks like. It uses this information to generate always up to date API documentation.

Get notified when your API changes

Since APIToolkit has an always up to date model of your API, you can get notifications and alerts immediately something changes. Eg an important field is no more, or a new field has been added, or it’s format changed from a US date format to an EU date format

Refactor and Migrate Safely

Migrate legacy with confidence. Don’t crash your apps because of fields which no one remembered to migrate.

Always up to date API docs

No more out of date Swagger files. You can upload your swagger to APIToolkit, & Download the autogenerated swagger based off your actual production traffic. Even generate the docs in different formats including Open API and RAML.
